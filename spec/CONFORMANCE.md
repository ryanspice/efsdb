# EFSDB Conformance Checklist

**Version:** 0.2.0

Implementations must pass these checks to be considered compliant.

## 1. Cryptography

- [ ] **KDF**: Correctly derives keys using `sodium_crypto_kdf_derive_from_key` with correct contexts (`manifest` padded to 8 bytes, `chunk` padded to 8 bytes) and IDs (2, 3).
- [ ] **Encryption**:
    - [ ] Writes the frozen envelope framing with suite IDs from `spec/enums/protection-suites.md`.
    - [ ] Uses random Nonces for every write.
    - [ ] Validates AEAD tags and nonces on decrypt for supported AEAD suites.
    - [ ] Preserves the frozen `H + 8 + N + C` contract where `payload_length = N` and the suite tag/checksum remains trailer `C`.
    - [ ] Fails if AAD does not match expected value (`manifest:$id`, `chunk:$hash`) for storage encryption and `0..H+7` for envelope suite `0x02`.
- [ ] **Master Key**: accepts 32-byte raw key from Base64 source.

## 2. Storage

- [ ] **Paths**:
    - [ ] Chunks: `/{entity}/chunks/{h1}/{h2}/{hash}.c`
    - [ ] Manifests: `/{entity}/manifests/manifest_{id}.m`
- [ ] **Atomicity**: Writes use `LOCK_EX` or atomic rename (implementation dependent, but must avoid partial writes).

## 3. Manifests

- [ ] **JSON**: Produces valid JSON with `v: 1`.
- [ ] **Fields**: Includes `id`, `entity`, `size`, `mime`, `mtime`, `chunks`.
- [ ] **Chunks**: Correctly lists all chunks in order with valid offsets and lengths.

## 4. Reading

- [ ] **Reassembly**: Can reassemble a file from chunks using the manifest.
- [ ] **Verification**: Verifies chunk hashes match the manifest during read (optional but recommended).

## 5. Interop

- [ ] **Fixtures**: Can decrypt and read the standard fixtures generated by the PHP reference implementation.
