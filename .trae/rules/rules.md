EFSDB is a PHP-first, multi-tenant control plane + site runtime. Public site stays at `/`; control plane lives under `/_efsdb`; no `?action=` routing. Prefer thin `index.php` route wiring with delegation to existing services. Routes/layouts are PHP runtime truth. Svelte custom elements and islands are enhancement UI only. Builder is separate from Explorer. Explorer is a power-user inspection surface, not the default editor. Editing happens in dedicated Svelte editor surfaces or windows. JSON is the structured content truth. Preview uses an iframe. Compile-on-save is the default workflow. Use `production`, not `published`. Do not hardcode machine-specific absolute paths. Active design authority lives in `web/ui/README.md`, `web/ui/design/efsdb-ui-design-brief-v0.1.md`, and the runtime token layer under `web/styles` plus `web/ui/design/tokens.ts`. Light mode remains first-class, dark mode remains the workstation baseline, and green is an optional accent rather than the default baseline.
