# Blake3 Extension Build & Test

The committed Blake3 PHP extension source of record lives at `efsdb/php/core/ext/blake3-src`. It is vendored from upstream commit `efb0ccdacf5c723090d60dd8eccef2a89547456a`, and the repository keeps both the upstream notes in `README.upstream.md` and the local reproduction notes in `README.md`. The locked Windows build path is `efsdb/php/core/ext/blake3-src/task.bat`, which runs `phpize`, `configure --with-php-build="..\deps" --enable-blake3 --with-mp="disable"`, and `nmake /nologo`. The committed Windows binary at `efsdb/php/core/ext/php_blake3.dll` was produced on Windows 11 Pro `10.0.22631` with Visual Studio Build Tools 2022 `17.14.27`. The extension compile flags are `/DZEND_ENABLE_STATIC_TSRMLS_CACHE=1 /DBLAKE3_NO_SSE2 /DBLAKE3_NO_SSE41 /DBLAKE3_NO_AVX2 /DBLAKE3_NO_AVX512`. Its SHA-256 is `371fba3c31f15e4ad74f6971ff5023fc3a9a36abf835a085f1d2c168df69d487`.

For Linux parity builds, the committed helper path is `efsdb/php/core/ext/blake3-src/build.sh`, which uses the vendored `Dockerfile` based on `ubuntu:24.04` to emit `compiled/blake3.so`. Development and CI entry points must load the repository `efsdb/php/core/php.ini`, and the integration assertion lives in `efsdb/php/core/bin/assert_repo_php_ini.php`. Any edit to this document is guarded in CI by a pull-request approval rule that requires two approvals before merge.
